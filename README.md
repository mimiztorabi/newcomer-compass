# Newcomer Compass — AI Product Portfolio Case Study

> **Portfolio project:** a fictional, static concept for a Senior Product Manager portfolio. It demonstrates product thinking around an AI-guided newcomer onboarding experience; it is **not** a production service.

Newcomer Compass helps a person who has recently moved to a city turn an overwhelming to-do list into a small, explainable next step. The included prototype is a browser-only conversation mockup: it does not call an AI model, collect personal data, or make recommendations from live sources.

## Start here

1. Read the [product brief](docs/product-brief.md) for the problem and opportunity.
2. Read the [PRD](docs/prd.md) for the proposed MVP scope.
3. Explore the static prototype by opening `prototype/index.html` in a browser, or host this repository with GitHub Pages.
4. Review the [case study](docs/case-study.md) for the portfolio narrative and placeholders to complete.

## Prototype notes

- **No external dependencies:** it runs with only a modern browser.
- **No backend or data storage:** selections remain in the current page session only.
- **No real AI:** sample responses are deliberately deterministic and labeled as a concept.
- **Safe to publish:** there are no API keys, trackers, forms, or paid services.

## What this demonstrates

This case study is organized so a reviewer can follow the product-management story in plain English:

- **Problem framing and strategy:** start with the product brief and its explicit target user, problem, outcomes, non-goals, and open questions.
- **Research and UX thinking:** use the research plan, user journey, and prototype to see what should be tested and why the flow starts with a simple choice.
- **AI interaction and responsibility:** review the AI behavior and risks documents for the proposed response contract, privacy boundaries, and escalation rules.
- **Trade-offs and measurement:** read the PRD, decision log, and evaluation plan to understand what this prototype deliberately does *not* do and how future work could be judged.

Every claim is either a fictional concept statement, an **Illustrative** example, or marked `TODO:` for the portfolio owner to replace with evidence.
