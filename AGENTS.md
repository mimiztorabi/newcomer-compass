## Purpose

This repository is a **public Senior Product Manager portfolio case study**, not a production application. Optimize for clarity, traceable product reasoning, and a safe static demo.

## Working guidelines

- Keep the prototype static and compatible with GitHub Pages.
- Do not add API keys, secrets, analytics, paid services, backends, databases, or external runtime dependencies.
- Mark invented facts, metrics, research, and outcomes as `TODO:` or `Illustrative` until the portfolio owner replaces them with evidence.
- Use plain English and explain AI decisions, uncertainty, safety boundaries, and evaluation methods.
- Keep visual assets small and place prototype screenshots in `assets/prototype-screens/`.
- Test the static prototype in a browser after changing its behavior or presentation.

## Documentation conventions

- Start documents with a short comment explaining their purpose.
- Prefer short sections, decision tables, and links between related documents.
- Keep user data examples fictional and non-sensitive.
